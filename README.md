4. 핵심 코드 및 배운 점

1.	Semantic 구조 잡기 (header, section, footer)
기존에 div만 쓰던 것에서 벗어나, 페이지 상단 소개 영역 → header, 핵심 콘텐츠(설명 + 입력 폼) → section, 저작권 안내 → footer 로 나누어 구조를 잡았다.
이 과정을 통해 콘텐츠 역할에 맞는 시맨틱 태그를 선택하는 법을 배웠고, 나중에 유지보수나 접근성, 검색엔진(SEO) 측면에서도 더 유리하다는 점을 이해했다.

2.	외부 리소스 연동 (CSS, 웹폰트)
<link rel="stylesheet" href="./styles/reset.css"> 와 <link rel="stylesheet" href="./styles/style2.css"> 를 통해 CSS를 HTML과 분리해서 관리하는 법을 사용했다.
Google Fonts에서 Noto Sans KR, Moirai One을 불러오는 코드를 사용하면서 웹폰트 적용 과정(사전 preconnect 설정, href 구성 등) 을 실제로 경험했다.
이를 통해 타이포그래피도 “디자인 자산”이라는 감각을 갖게 되었다.

“훈련하러 가기 GO! GO!” 버튼 클릭 시 모달이 열리고, 모달의 “종료하고 진짜 훈련하러 가기 GO!GO!” 버튼을 누르면
모달이 닫히면서 특정 페이지나 섹션으로 이동하도록 JavaScript 이벤트를 추가할 예정이다.
이 과정에서 transition·opacity 등을 활용해 부드러운 등장/퇴장 애니메이션도 적용해 보고 싶다.
현재 header 안에 <width: 221.604px;></width:>, <height: 225px;></height:>
<flex-shrink: 0;></flex-shrink:> 등 유효하지 않은 태그가 들어있다. 

6. 개발하며 느낀 점
강사님이 얘기주신 div로 box를 그린 다음 radius를 50% 줘야되고 백그라운드 컬러를 주지 않았다.  그 안에 저 로고들을 집어넣지 않았다.  
“그냥 보는 HTML”에서 “내가 설계하는 구조”로 시야가 바꼈다.
잘못된 태그(<width: …>)를 넣었을 때 브라우저는 그냥 무시하지만, 코드를 다시 보면서 “아, 이런 속성들은 CSS로 가야 한다”는 걸 몸으로 배웠다. 
앞으로는 HTML은 구조, CSS는 디자인, JS는 동작이라는 역할 분리를 더 의식하게 될 것 같다. 
사용자 경험(UX)을 자연스럽게 떠올리게 된 계기 단순한 페이지지만 여전히 css 구조가 어렵다. 
“이 버튼을 누르면 어떤 느낌이어야 하지?” “결과를 보고 나서 사용자가 기운이 났으면 좋겠다.” 라는 생각이 들었다. 
그래서 응원 문구와 라이캣 이미지를 넣으면서, 코드가 결국 사람의 감정을 움직이는 도구라는 걸 조금 더 실감하게 되었다.
